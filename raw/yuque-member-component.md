# 套餐会员组件逻辑

> 来源：语雀 - 埋铭的知识/项目文档
> URL: https://aliyuque.antfin.com/fqicv9/kg7h1z/edesekqz30582ck7

> 摘要：套餐会员组件产品逻辑，包含会员组件露出规则、字段获取、会挑逻辑、交易打标流程

## 需求范围
- 类目：在线预约套餐
- 商家：参与会挑的商家目前支持希尔顿，后续有可能新增，由业务商品标控制
- 会员组件样式调整针对类目全量商品

## 业务逻辑
- 会挑商品必有snp，会挑和一住升金互斥，会挑商品必为会员专享
- 会挑、一住升金、会员专享都是item id维度

## 功能点
| 域 | 改动点 |
|------|------|
| 套餐发布 | 会挑商品识别；商品是否会员专享、是否享会员权益配置（现成能力）。用ic打标实现 |
| 套餐c端 | snp配置展示；会挑商品打标+传参落库；商详/下单/二次预约下单会员组件调整为消费平台会员数据；by uid+item id消费玩法数据展示会挑进度；是否参与会挑落订单快照 |
| 酒店交易 | 识别会挑商品ic标，在二次预约订单下单打tccp标识供玩法识别；逆向无需改动 |
| 玩法 | 识别酒店交易参数写入会挑进度（现成能力）；逆向无需改动 |
| 会员 | 支持套餐入参uid和seller信息，获取平台会员的一通百组件信息 |

## C端样式改动

### 会员组件露出逻辑
涉及的位置：
- 套餐商品详情页、套餐第一单下单页（即买即约+先囤后约）
- 二次预约日历下单页、二次预约日历订单详情【酒店日历】

规则：
- 商品享会员权益的情况下，注册组件、会员组件，本次均由平台会员侧返回
- 商品不享会员权益的情况下，期望取会员组件返回字段自行拼接前端样式

### 会员组件字段获取
| 场景 | 区块 | 逻辑 |
|------|------|------|
| 商品享会员权益（复用平台会员组件） | 会员表达（一通百）+会挑 | 复用平台现有组件，但会挑有逻辑定制：是否透出会挑由套餐自行做一道前置判断——当用户已开通会员时，消费套餐上是否含有ic tag:xxx，有则调用会员/玩法既有会挑信息透出 |
| 同上 | 激活/开通引导 | 复用平台 |
| 同上 | 权益 | 完全复用平台现有组件及跳转 |
| 同上 | 运营配置活动 | 定制，在套餐商品上屏蔽不透出 |
| 商品不享受会员权益（且必然无会挑） | 会员组件 | 已开通已激活文案套餐侧按线上处理；其他透出的卡片组件、一通百文案、激活入口、开通入口由平台下发 |

### 二次预约下单/订单的会员组件
参考上述逻辑，取**套餐第一单item id上的信息**透出渲染会员组件，而非日历单对应的rate id信息。

## 交易流程：套餐预约订单识别为会挑
- 二次预约日历单打标：
  - 打标场景：套餐商品含ic tag=xxx的二次预约日历下单，创tccp单打标
  - 打标参数：只打tccp标，k=hotelShowMemberRight
- 套餐单（P0）落订单快照：
  - 范围：只针对ictag是会挑商品的增加订单快照字段
  - 字段：落在套餐信息 → 新增字段【是否参与会员挑战】→ 是
