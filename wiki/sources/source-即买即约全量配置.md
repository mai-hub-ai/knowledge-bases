# 即买即约支持商品自动化全量配置 摘要

## 文档信息
- **来源**: 钉钉文档
- **URL**: https://alidocs.dingtalk.com/i/nodes/NZQYprEoWoDjdZb1IP1ZO0KRV1waOeDk
- **类型**: 补充需求（准入规则变更）

## 需求背景
即买即约已全量生效，原白名单靠运营人工维护存在滞后性且易出错，需支持全量配置，for后续日常化运营。

---

## 核心变更：准入机制

### 变更前（白名单机制）
- 商品需在 sellerid + itemid 的白名单中
- 需配置至少1个专属权益

### 变更后（黑名单机制）
- **默认支持**：在线预约在售的**国内套餐商品**均支持即买即约
- **黑名单拦截**：在黑名单里的默认不支持即买即约

---

## 支持范围

### 类目范围
- **仅在线预约类目**

### 地域范围
- **仅支持国内**（国际支持后同步放开）
- 国际判断标准（未测试先拦截）：
  1. citycode > 70w
  2. 国际卖家：带有标 hk
  3. 外币结算

### 商品状态要求
- 商品需**上架状态**
- 未上架、定时上架未到时间、已下架或售罄item不支持

---

## 不支持即买即约的商品类型（系统过滤）

| 类型 | 说明 |
|------|------|
| 国际套餐商品 | 暂时功能不支持，后续同步放开 |
| 配置出行人组件商品 | - |
| 酒+酒的商品 | - |
| ebk快速发品 | - |
| 定金预售 | **新增** |
| 未到预约有效期商品 | **新增**：当前时间早于item的预约开始时间 |
| 日历预订的品 | **新增** |

---

## 权益生效渠道

### 配置规则
- 支持业务配置单个 item + 即买即约 或 单个 item + 先囤后约
- 优先级：业务配置 > 兜底规则
- 兜底规则：默认"先囤后约和即买即约渠道均生效"

### 发布端文案
支持业务配置权益生效渠道文案

---

## 下单页Tab文案

- 支持业务配置 itemid + 文案
- 未配置时兜底文案：**"火爆抢约 早订安心"**

---

## 业务注意事项

> ⚠️ **重要**：全量后，即买即约生效时不校验商品实际的价库，即无法保证用户整份预约时日历页一定有库存。

**场景举例**：
- 某SKU住2晚可拆分，但仅有1晚价库，无法连住2晚
- 此时系统无法识别，用户端会渲染日历约满

---

## 提取的变更点
1. 准入机制：白名单 → 黑名单
2. 新增不支持类型：定金预售、未到预约有效期商品、日历预订的品
3. 权益配置可选（非必需）
4. 下单页Tab支持自定义文案
