# 业务语义字典

## 定义
业务语义字典定义了酒店套餐产品中常用的业务术语和概念，确保产品、开发、运营对术语的理解一致。

---

## 核心术语

### 单店 vs 通兑

| 类型 | 定义 | 判断依据 | 说明 |
|------|------|---------|------|
| **单店** | item（商品）下只挂了**1个**酒店（shid）的商品 | shid数量 = 1 | 单一酒店套餐 |
| **通兑** | item下挂了**>=2个**酒店的商品 | shid数量 >= 2 | 多酒店通用套餐 |

**关键区分点**：
- 判断依据：item关联的shid数量
- 不是按SKU数量区分，而是按关联的酒店数量区分

#### 示例

```
商品A（单店）:
- item_id: 1001
- 关联shid: [酒店X]
- 判定: shid数量=1 → 单店

商品B（通兑）:
- item_id: 1002
- 关联shid: [酒店X, 酒店Y, 酒店Z]
- 判定: shid数量=3 → 通兑
```

## 业务影响

单店/通兑的区分影响多个业务逻辑：

| 业务场景 | 单店逻辑 | 通兑逻辑 |
|---------|---------|---------|
| 库存管理 | 单一酒店库存 | 多酒店库存聚合 |
| 价格展示 | 单一价格 | 价格区间或最低价 |
| 节假日可住 | 直接判断 | 需计算：满足节假日可住的hid总数/关联hid总数 > 99% |
| 标签透出 | 直接透出 | 可能需聚合判断 |
| 可住日历 | 展示 | 不展示（详见 [[日历组件]]） |
| 可约酒店页 | 不展示 | 展示（详见 [[套餐选择区]]） |
| 酒店榜单 | 获取 | 不获取（详见 [[榜单标签]]） |

## 售中-套餐相关玩法功能

> 来源：钉钉文档「售中-套餐相关玩法功能说明」文件夹截图
> 摄入时间：2026-04-16

### 补差升房 / 免费升房

| 维度 | 说明 |
|------|------|
| **定义** | 原绑定房型约满后，用户可加价升级至更高房型 |
| **触发条件** | 原绑定基础房型**约满**时展示（免费升房除外） |
| **免费升房** | 即补差 0 元升级房型，基础房型**未约满时也会展示** |
| **价格约束** | 补差金额需对比日历房价差有优惠（≤9 折），且不能超过单 SKU 单间夜价格的 **40%** |
| **管控规则** | 禁止商家恶意关房引导用户预约更贵房型，平台基于竞对房态对比进行巡检管控 |

### 预约搭赠

| 维度 | 说明 |
|------|------|
| **定义** | 在线预约促核销场景中，预约后向用户发放额外权益（搭赠玩法） |
| **权益类型** | ① 核销后返现：FP 现金红包 → 飞猪钱包，可直接提现 |
|  | ② 预约/核销后赠优惠券：通过优惠平台或商家创建，发放至用户卡包，购买指定商品时立减 |
| **适用场景** | 大额券促核销预约、小额券针对已约用户做店内商品二次复购、日历房优惠券促再次预订复购 |

### 套餐延期

| 维度 | 说明 |
|------|------|
| **定义** | 大促高峰期售卖订单多但履约周期有限时，商家可申请延长套餐有效期 |
| **典型场景** | 618 售卖的套餐有效期仅到 0830，3000 待预约订单 → 可延期至 12.31 继续预约 |
| **核心价值** | 避免大量未预约订单到期自动退款，让囤货用户有更充裕的预约时间 |

### 跨 SKU / 跨 Item 预约

| 维度 | 说明 |
|------|------|
| **定义（同 item 跨 SKU）** | 囤了某档 SKU 的用户，预约时可**补差预约**同 item 内高价位 SKU 的酒店&房型，或**退差预约**低价位 SKU 的酒店&房型 |
| **生效条件** | ① 套餐单晚金额可整除（1 晚/多晚不可拆分不受影响，多晚可拆分需每晚金额可整除）② 被跨套餐支付订单 > 10 |
| **商品范围** | 仅支持国内在线预约套餐 |
| **配置方式** | 无需酒店&商家新增配置，提供 item id 加入白名单即可（后续预计默认开启） |

### 申请库存（候补）

| 维度 | 说明 |
|------|------|
| **定义** | 用户想约日期约满时，可线上发起入住日期申请，替代联系客服协商 |
| **核心价值** | 缩短客服协商时效、降低流失，用用户确定性订单换取商家确定性库存 |
| **商品规则** | 全量放开（非直连 & 直连套餐均适用） |
| **日期规则-可触发** | ① rate + 日期房态开，但可约库存 = 0（已约满）② rate + 日期房态关，但飞猪日历房正常在售 |
| **日期规则-不可触发** | 商品发布时设置的不可住日期 |
| **频率限制** | 未来实际可约天数占比 ∈ (20%, 99%)；单酒店（hid）下待确认申请订单 < 3 |
