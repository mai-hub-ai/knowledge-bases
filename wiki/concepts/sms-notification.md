# 短信通知

## Why This Exists
短信通知是套餐订单系统的底层能力之一，贯穿用户从购买到退款的完整订单生命周期。其核心价值有二：1）保障用户订单信息透明，降低客诉；2）通过各触点引流用户至微信私域（福利官/管家），提升复购和服务体验。

## Definition
套餐短信通知体系由**订单生命周期通知**和**营销引流通知**两大类组成，通过短信模板+事件驱动机制，在订单关键节点自动发送短信给用户。

## 触发系统架构

短信发送分布在 **5 个系统** 中：

| 系统 | 职责 | 典型场景 |
|------|------|---------|
| **triptuan** | 套餐核心业务，处理购买/预约/退款事件 | 购买提醒、预约提醒、退款提醒 |
| **hotel-fc** | 酒店履约中心，处理在线预约相关事件 | 过期自动退款、购买成功、可约提醒 |
| **tuan-trade** | 套餐交易系统，处理支付/代扣事件 | 0元囤代扣、澳门通支付 |
| **tuan-task** | 套餐定时任务，处理营销引流 | 频道页引流（批量发送） |
| **tuan-gateway** | 套餐网关，处理商家操作事件 | 商家确认凭证、可约提醒 |

### 双触发点场景

以下场景在两个系统中均有触发点，需注意**去重逻辑**：
- **购买提醒**: triptuan `EnableNotifyListener` + hotel-fc `OnlineBookingPaidSuccSmsTaskExecutor`
- **可约提醒**: tuan-gateway `OnlineBookingSmsNoticeTaskExecutor` + hotel-fc 同名类

## 短信内容模式

### 通用结构
所有短信内容遵循统一模式：
```
[事件通知] + [引流链接] + 拒收请回复R
```

### 引流链接
| 短链 | 用途 |
|------|------|
| https://a.feizhu.com/0oICBj | 通用福利官/顾问微信 |
| https://a.feizhu.com/0ZYL5o | 行程管家微信 |
| https://a.feizhu.com/0Kcern | 大促福利官 |
| https://a.feizhu.com/4QDqif | 套餐管家微信 |

### 动态变量
| 变量 | 含义 |
|------|------|
| `${item_name}` | 套餐商品名称 |
| `${checkin}` | 入住日期 |
| `${checkout}` | 离店日期 |
| `${n}` | 入住晚数 |

## Constraints & Edge Cases
- **4个引流场景未配置**（编号12/14/16/17），存在引流触点空白
- 引流类短信统一使用模板 724605350，旧模板（380763747、235103410）已废弃
- 退款成功提醒（310213555）被复用于"商家确认电子凭证预约"场景，内容实际是预约成功确认
- 0元囤代扣短信为独立流程，不走 triptuan 主链路
