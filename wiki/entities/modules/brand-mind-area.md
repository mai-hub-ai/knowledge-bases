# 会员组件区

> 原名：品牌心智区，已更名为会员组件区

## 概述
展示酒店品牌会员相关信息（SNP、会挑、一住升金、会员专享等），建立用户对套餐/酒店的信任感并引导会员转化。会员组件数据由**平台会员侧**统一下发，套餐侧在部分场景做前置判断和定制。

## 核心属性

| 属性 | 值 |
|------|-----|
| 模块序号 | 3 |
| 模块类型 | 会员展示模块 |
| 适用类目 | 在线预约套餐（全量商品） |
| 数据来源 | 平台会员侧返回 + 套餐侧前置判断 |
| 文档状态 | ✅ 已补充 |

## 业务概念关系

| 概念 | 说明 | 互斥/依赖 |
|------|------|-----------|
| **会挑**（会员挑战） | 挑战进度条，累积住宿晚数升级会籍 | 会挑 ↔ 一住升金 **互斥** |
| **SNP** | 酒店品牌会员积分/权益标识 | 会挑商品**必有** SNP |
| **一住升金** | 单次入住即升金卡 | 与会挑互斥 |
| **会员专享** | 会员专享价格标识 | 会挑商品**必为**会员专享 |

- 以上标识均为 **item id 维度**
- 会挑商品由业务通过 ic tag 打标控制，目前支持希尔顿，后续可新增商家

## 组件区块与逻辑

### 场景一：商品享会员权益（复用平台会员组件）

| 区块 | 逻辑 |
|------|------|
| **会员表达（一通百）+ 会挑** | 复用平台组件，但**会挑有定制**：套餐侧做前置判断——当用户已开通会员 → 检查套餐是否含 ic tag → 有则调用会员/玩法既有会挑信息透出 |
| **激活/开通引导** | 完全复用平台组件 |
| **权益** | 完全复用平台现有组件及跳转 |
| **运营配置活动** | **定制**：在套餐商品上屏蔽不透出 |

### 场景二：商品不享受会员权益（且必然无会挑）

| 区块 | 逻辑 |
|------|------|
| **会员组件** | 已开通已激活文案，套餐侧按线上处理；其他透出的卡片组件、一通百文案、激活入口、开通入口由平台下发 |

### 特殊场景：二次预约下单/订单

- 取**套餐第一单 item id** 上的信息透出渲染会员组件
- **不取**日历单对应的 rate id 信息
- 会挑判断基于套餐 item 的 ic tag

## 会挑交易打标流程

### 二次预约日历单打标
- **打标场景**：套餐商品含 ic tag = xxx → 二次预约日历下单 → 创 tccp 单打标
- **打标参数**：只打 tccp 标，`k=hotelShowMemberRight`

### 订单快照
- **套餐单（P0）**：仅针对 ic tag 为会挑商品 → 套餐信息新增字段「是否参与会员挑战」= 是
- **二次预约日历单（P2）**：暂不投入

---

## 会挑进度规则（希尔顿示例）
- 挑战进度条仅累积在飞猪预订的**含会员权益的日历房晚**和**指定含 SNP 套餐房晚**
- 会员在 **120 天**挑战期内完成 **8 晚**及以上住宿 → 挑战成功 → 保留金会籍至下一个日历年结束
- 否则挑战失败 → 保留银会籍至下一个日历年结束
