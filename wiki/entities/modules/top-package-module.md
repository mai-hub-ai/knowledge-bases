# 置顶套餐模块

## 概述
置顶套餐模块是酒店日历页（小搜detail页）顶部的套餐展示模块，用于优先展示高转化套餐商品。

## 模块位置
- **页面**：酒店日历页（小搜detail页）
- **位置**：页面顶部

---

## 透出优先级

酒店日历页套餐模块的透出顺序：

```
待预约套餐 > 置顶套餐 > G35 > 酒x > 日历房最低价
```

| 优先级 | 模块 | 说明 |
|--------|------|------|
| 1 | 待预约套餐 | 用户已有待预约订单 |
| 2 | **置顶套餐** | 本模块 |
| 3 | G35 | - |
| 4 | 酒x | - |
| 5 | 日历房最低价 | - |

## 圈品池配置

### 类目范围
在线预约类目

### 端范围
淘宝、飞猪、微信、支付宝、H5

### 商品池分类

| 池子 | 说明 | 用途 |
|------|------|------|
| 裸房池 | itemid1、itemid2 | vs日历比价比例100% |
| 含X元素池 | itemid3、itemid4 | vs日历比价比例130% |

> ⚠️ **业务注意**：系统不做是否是裸房或含X的判断，业务需自行保障两个池子的itemid准确性、不重复。

## 圈品逻辑

### 1. shid & skuid匹配

1. 基于小搜传入圈品id，获取业务圈选的itemid list
2. 基于小搜传入的shid、业务圈选的itemid，进行关联关系匹配
3. 每个itemid下，只选择关联的**最低价skuid**（summary券后价最低价）
4. 若当前shid无任一关联的skuid，则返回无数据

### 2. 透出判断

#### 可售卖判断
| 条件 | 说明 |
|------|------|
| 售卖期 | itemid上架在售状态 |
| 库存 | sku剩余可售库存 >= 1 |
| 份数诉求 | 用户入住晚数为sku晚数整数倍，且 <= sku剩余可售份数 |

**计算公式**：
```
需购买份数 = cico连住晚数 * 房间数 / sku入住晚数
需购买份数 <= sku剩余可售库存数
```

#### 价库要求判断

在listing页开始遍历即买即约日历页，落缓存数据：

| 参数 | 说明 |
|------|------|
| shid | 酒店ID |
| skuid | 套餐SKU ID |
| srid | 套餐关联的rate ID |
| cico | 入住离店时间（当前时间往后15天/7天） |
| 房间数 | 用户选择的房间数 |
| nop | 人数（1或2） |

**输出数据结构**：
```
shid + skuid1、rateid1、即买即约总价1、sku券后价、裸房/含x
shid + skuid2、rateid2、即买即约总价2、sku券后价、裸房/含x
...
```

## 导购价格校验逻辑 ⚠️

> **重要**：套餐回传多个sku id后，导购需要进行套餐vs日历房比价。

### 比价逻辑

| 商品池 | 比价阈值 | 说明 |
|--------|----------|------|
| 裸房池 | 不lose日历100% | 套餐价格 <= 日历房价格 |
| 含优质X池 | 不lose日历130% | 套餐价格 <= 日历房价格 * 1.3 |

### 价格取值

| 价格类型 | 取值说明 |
|----------|----------|
| **套餐价格** | 由套餐传入（即买即约总价） |
| **日历房价格** | srid+早餐数条件+房间数条件下，C价总价最低 |

### 特殊情况枚举

| 情况 | 处理方式 |
|------|----------|
| 含早套餐找不到对应日历rate | 视为套餐beat，可透 |
| 不含早套餐找不到对应日历rate | 比同一srid下最低C价，beat 100%/130%才透出 |
| 套餐/日历部分含早 | 取cico区间内含早份数最多一天的早餐份数匹配比较 |

### 套餐含早数定义

| 商品类型 | 含早数取值 |
|----------|------------|
| 在线预约_先囤后约 | 每晚含早数相同，取该值 |
| 在线预约_日历预订（每晚安数相同） | 每晚含早数相同，取该值 |
| 在线预约_日历预订（每晚含早数不同） | 取cico区间内含早份数最多一天的早餐份数 |

### 多sku择优逻辑

套餐取回的sku在过掉价格校验后如果还有多个：
1. 取**sku券后价最低**的
2. 相同则随机取

## C端链路交互

### 从置顶坑位进入商品详情页

| 场景 | 行为 |
|------|------|
| 进入商品详情页 | 唤起SKU加购浮层，选中对应SKU |
| 购买份数计算 | （小搜房间数 × cico连住晚数）/ sku入住晚数，无法整除则置为1 |
| 点击购买 | 跳转即买即约下单页，锚定立即预约tab |

### 即买即约下单页交互

| 步骤 | 行为 |
|------|------|
| 进入下单页 | 锚定立即预约tab，唤起即买即约日历浮层 |
| 通兑商品 | 跳过选酒店列表页，渲染对应shid的日历页 |
| 选中日期 | 选中srid、间晚数组合（来自detail页传入） |
| 日期不可约 | 日期默认不选中，toast提示"当前选择的日期不可约，请更换入住日期" |
| 点击下一步 | 渲染即买即约下单页，走即买即约下单 |

### 中间跳出场景

| 行为 | 说明 |
|------|------|
| 更换SKU或关闭浮层 | 商品详情页支持即买即约功能，点击查看日期进入日历页 |
| 再次点击购买 | 默认跳转即买即约下单页，detail页参数清空 |

## 权益表达规则

| 场景 | 权益表达 |
|------|----------|

| 页面 | 非白名单商品 | 白名单商品 |
|------|--------------|------------|
| 商品详情页心智区 | 同先囤后约表达 | 即买即约权益表达 |
| 即买即约日历页 | 普通预约日历页权益表达 | 即买即约权益表达 |
| 即买即约下单页 | 日期命中则透出权益 | 即买即约权益表达 |
