# 囤好货标签区

## 概述
囤好货标签区展示套餐的核心购买优势（先囤后约、不约可退、过期可退），并提供可住日历功能帮助用户快速了解可预约日期。是促进用户决策的关键模块。

## 核心属性

| 属性 | 值 |
|------|-----|
| 模块序号 | 7 |
| 模块类型 | 营销标签模块 |
| 位置 | 服务标签区下方 |
| 文档状态 | ✅ 已补充 |

### 浮层解释文案

| 文案 | 解释 |
|------|------|
| **先囤后约** | 先下单锁定优惠，确定出行日期后，在线预约具体出行日期。 |
| **不约可退** | 若您的整份宝贝完全未进行预约，支持随时发起100%退款，整份宝贝部分预约则不支持退款。 |
| **过期可退** | 商品过期后您的整份宝贝仍未预约，系统自动100%退款，整份宝贝部分预约则不支持退款。 |

## 可住日历

### 透出条件

⚠️ **重要限制**：仅**单店&单sku**才透出可约日历

| 场景 | 是否透出 | 说明 |
|------|---------|------|
| 单店 + 单sku | ✅ 透出 | 当前item在售sku=1 |
| 单店 + 多sku | ❌ 不透出 | 当前item在售sku>1 |
| 通兑 | ❌ 不透出 | item关联>=2个shid |

**单sku判断**：当前item在售sku=1

### 单店可住日历逻辑

**展示规则**：
1. 基于item的可住日期范围
2. 选择在售的**最低价sku** + **第1个可约房型** + 默认间晚数
3. 从【最近可约的日期开始】展示
4. 外露x天

**点击跳转**：
- 跳转到即买即约日历页
- 锚定：最低价sku、第一个可约房型
- 暂不锚定具体入离日期
- 大促即买即约降级走到普通日历页

### 外露天数规则

| 场景 | 外露天数 | 日历类型 |
|------|---------|---------|
| 日常期 | 30天 | 即买即约日历页 |
| 大促期间（峰值） | 30天 | 普通日历（即买即约降级） |

> 对标抖音外露30天

### 日历条样式

**信息结构**：

```
┌─────────────────┐
│  头部行：今天/明天/法定假日（支持显示"休"）  │
│  中间行：日期数字                           │
│  底部行：可约/约满/紧张 + 加价文案          │
└─────────────────┘
```

**样式规则**：
- 字色见设计稿
- 是否置灰：由是否可约决定
- 升房/协商/预约有礼：不在外层表达

### 日历页样式

- 即买即约日历页当前样式符合预期
- 旧版日历页：将【紧张】文案下发为红色
- 确认【可约/约满/加价】【今/明/节假日】【休字】的字色与即买即约日历页对齐

### 通兑逻辑

- 若item=通兑，则**不展示**可住日历

详见 [[业务语义字典]] 中"单店 vs 通兑"的定义
