# 商品评价区

## 概述
展示商品评价和酒店评价两个维度的评分评价信息，支持单店/通兑差异化展示，含评价排序、跳转锚点、无评价兜底等完整逻辑。

## 核心属性

| 属性 | 值 |
|------|-----|
| 模块序号 | 12 |
| 模块类型 | 信息展示模块 |
| 文档状态 | ✅ 已补充 |

---

## 评价双维度结构

| 维度 | 数据来源 | 展示条数 |
|------|---------|---------|
| **商品评价** | 调用评价接口获取 | 默认展示 **2 条**优质评价 |
| **酒店评价** | 调用评价接口 + 套餐侧酒店排序接口 | 单店 **1 条** / 通兑 **2 个酒店** |

---

## 商品评价

- 默认展示 2 条优质评价
- 评价维度：描述相符 / 住宿环境 / 服务态度 / 性价比
- 点击跳转评价列表页，自动锚定到【商品评价】TAB

## 酒店评价

### 单店场景
- 展示最多 **1 条**优质评价
- 调用评价接口，传入 **SHID** 获取
- 展示：酒店名称 + 评分 + 评价量
- 点击直接到达此酒店评价详情页

### 通兑场景
- 展示最多 **2 个酒店**
- 调用套餐侧**酒店列表排序接口**获取酒店排序
- 展示排序最前城市中排序最前的 2 个酒店的：名称 + 地理信息 + 评分 + 评价量
- 点击酒店评价：拉起所有酒店的评价列表页
- 点击具体酒店：拉起该酒店评价详情浮层

### 通兑特殊处理
- 当排序最前的 2 个酒店中，**任一酒店评分 = 0 或评价数 < 5**
- → 酒店评价模块**不展示**具体酒店
- → 仅保留：【酒店评价 共XX家酒店 查看全部XX条 >】

---

## 评价列表页排序规则

### 单店 / 通兑具体酒店下的评价
- 由评价侧计算排列

### 通兑酒店排序
- 评价侧调套餐侧酒店列表排序接口获取城市排序 + 酒店排序
- 切换筛选不同城市时，酒店排序仍生效

**排序优先级**：
1. 按**热门程度**排序城市，默认选中最热门城市（运营维护配置）
2. 不同城市的酒店中，优先展示**可约的酒店**
3. 可约酒店中，优先展示**高人气酒店**
4. 均为高人气酒店 → 混排

### 高人气酒店计算规则（T+1 更新）
| 条件 | 说明 |
|------|------|
| 酒店评分 > X 分 | 评分阈值 |
| 近一年好评率 > X% | 好评率阈值 |
| 评价条数 > YY 条 | 评价量阈值 |

> 标签下线条件：评分低于 X 分 或 近一年好评率低于 XX%

## 无评价兜底

| 场景 | 展示 |
|------|------|
| 只有商品评价 | 展示商品评价，酒店评价区域展示「暂无评价」 |
| 只有酒店评价（单/多酒店） | 展示酒店评价，商品评价区域展示「暂无评价」 |
| 都无评价 | 均展示「暂无评价」 |

## 跳转锚点汇总

| 点击区域 | 跳转目标 | 锚定 TAB |
|---------|---------|---------|
| 商品评价区域 | 评价列表页 | 商品评价 |
| 酒店评价区域 | 评价列表页（通兑：所有酒店 / 单店：该酒店详情） | 酒店评价 |
| 具体酒店卡片 | 通兑：该酒店评价详情浮层 / 单店：该酒店评价详情页 | 酒店评价 |
| 单店基础信息评价模块 | 评价列表页 | 酒店评价 |
