# 日历组件

## 概述
日历组件是套餐详情页的决策核心模块，展示可住日期和可住日历，帮助用户了解可预约的日期范围和状态。本页是**可住日历规则的唯一权威来源**。

## 核心属性

| 属性 | 值 |
|------|-----|
| 模块序号 | 9 |
| 模块类型 | 决策核心模块 |
| 文档状态 | ✅ 已补充 |

## 一、可住日期规则

- **计算方式**: 取 item 可住日期 + 提前预约天数
- **文案**: 和现在保持一致

---

## 二、可住日历

### 透出条件

⚠️ **重要限制**：仅**单店**才展示可住日历

| 场景 | 是否透出 | 说明 |
|------|---------|------|
| 单店 | ✅ 透出 | item 关联 1 个 shid |
| 通兑 | ❌ 不透出 | item 关联 >= 2 个 shid |

详见 [[业务语义字典]] 中"单店 vs 通兑"的定义

---

### 单店可住日历展示逻辑

**基础规则**：基于 item 的可住日期范围，选择在售的**最低价 SKU** + **第 1 个可约房型** + 默认间晚数下，从**最近可约的日期开始**展示，外露 x 天。

| 维度 | 规则 |
|------|------|
| SKU 选择 | 在售的最低价 SKU |
| 房型选择 | 第 1 个可约房型 |
| 间晚数 | 默认间晚数 |
| 起始日期 | 最近可约的日期 |
| 外露天数 | x 天（见下表） |

### 外露天数规则

| 场景 | 外露天数 | 日历类型 | 说明 |
|------|---------|---------|------|
| 日常期 | 30 天 | 即买即约日历页 | 对标抖音外露 30 天 |
| 大促期间（峰值） | 30 天 | 普通日历数据（即买即约降级） | 即买即约会降级 |

---

### 点击跳转逻辑

| 维度 | 说明 |
|------|------|
| **跳转目标** | 即买即约日历页 |
| **锚定 SKU** | 最低价 SKU |
| **锚定房型** | 第一个可约房型 |
| **锚定日期** | 暂不锚定具体入离日期 |
| **大促降级** | 即买即约降级时走到普通日历页 |

## 三、日历条样式

### 信息结构

每个日期单元格由三行组成：

```
┌─────────────────────────────────┐
│  头部行：今天 / 周几 / 法定假日  │  ← 优先级：法定假日 > 今天 > 周几
│          + "休"独立展示          │
│  中间行：日期数字               │
│  底部行：预约状态文案 + 加价文案  │  ← 可约 / 约满 / 紧张
└─────────────────────────────────┘
```

### 头部行优先级

| 优先级 | 内容 | 说明 |
|--------|------|------|
| 1（最高） | 法定假日名称 | 如"元旦"、"春节" |
| 2 | "今天" | 当天日期 |
| 3（最低） | 周几 | 如"周一"、"周日" |

- **"休"字**：独立展示，与上述内容并存
- 逻辑与即买即约日历页内逻辑一致

### 样式规则

| 规则 | 说明 |
|------|------|
| 字色 | 见设计稿 |
| 是否置灰 | 由是否可约决定 |
| 升房/协商/预约有礼 | **不在外层表达** |

| 今 / 明 / 节假日 | 字色与即买即约日历页对齐 |

## Constraints & Edge Cases
- 通兑 item（shid>=2）**不展示**可住日历
- 外露天数日常期和大促期均为 30 天，但大促期使用普通日历数据（即买即约降级）
- 点击日历跳转时暂不锚定具体入离日期
- 升房/协商/预约有礼等信息不在日历条外层表达
- 旧版日历页的样式需与即买即约日历页对齐（尤其是"紧张"文案颜色）
