# 先囤后约下单页

## 概述
先囤后约下单页是传统二次预约套餐的购买路径，用户购买套餐凭证后，后续再预约入住日期。这是**最为常见**的下单页类型。

## 核心特征

| 属性 | 值 |
|------|-----|
| 订单生成 | 1笔套餐订单 |
| 支付方式 | 单独支付套餐金额 |
| 库存锁定 | 购买后锁凭证，预约时锁库存 |
| 适用人群 | 长期规划型用户 |

## 入口

- 商品详情页 - 购买按钮
- 商品详情页加购浮层 - 立即购买
- 即买即约双Tab下单页 - 先囤后约Tab

## 页面内容

以套餐下单页为框架，包含以下模块：

1. **套餐信息**：SKU名称、价格、数量
2. **联系人信息**：姓名、手机号
3. **会员组件**：会员商品展示注册组件
4. **当单可享**：优惠信息展示
5. **价格bar**：套餐价格明细

## 与即买即约的区别

| 维度 | 先囤后约 | 即买即约 |
|------|----------|----------|
| 订单数量 | 1笔 | 2笔（套餐+日历） |
| 日历组件 | 无 | 有 |
| 补差金额 | 无 | 有 |
| 专属权益 | 不享受 | 享受 |
| 预售支持 | 支持 | 不支持 |
| 购物车合并 | 支持 | 不支持 |
