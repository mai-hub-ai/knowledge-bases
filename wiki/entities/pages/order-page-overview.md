# 下单页概览

## 下单页分类

套餐下单页根据购买模式和商品类型分为四种：

| 类型 | 说明 | 常见程度 |
|------|------|----------|
| [[先囤后约下单页]] | 传统二次预约套餐的购买路径 | 最为常见 |
| [[即买即约双Tab下单页]] | 支持即买即约商品的混合下单页 | 新增 |
| [[日历预订下单页]] | 日历套餐的预订下单页 | 较少 |
| [[境内外酒店套餐下单页]] | 境外商品的特殊下单页 | 国际场景 |

---

## 下单页与购买模式的关系

### 先囤后约 vs 即买即约

这两种模式可以出现在**同一个商品**里：

| 维度 | 先囤后约 | 即买即约 |
|------|----------|----------|
| 购买路径 | 购买套餐凭证 → 后续预约日期 | 购买时直接选择入住日期 |
| 订单生成 | 生成1笔套餐订单 | 同时生成2笔订单（套餐单+日历单） |
| 支付方式 | 单独支付套餐金额 | 合并支付（套餐金额+补差金额） |
| 库存锁定 | 购买后锁凭证，预约时锁库存 | 购买时直接锁定库存 |
| 适用人群 | 长期规划型用户 | 即时决策型用户 |

### 入口判断

- **支持即买即约的商品**：展示双Tab下单页（立即预订 + 先囤后约）
- **不支持即买即约的商品**：仅展示先囤后约下单页

---

## 即买即约双Tab下单页结构

当商品支持即买即约时，下单页展示双Tab：

```
┌─────────────────────────────────────┐
│  [立即预订]    |    [先囤后约]       │  ← Tab切换
├─────────────────────────────────────┤
│                                     │
│  立即预订Tab内容：                   │
│  - 酒店信息模块（入离日期、酒店名称）│
│  - 入住人组件                        │
│  - 预订信息（房间数、入住人、手机号）│
│  - 当单可享组件（合并套餐+日历）     │
│  - 价格bar（套餐金额+补差金额）      │
│                                     │
│  先囤后约Tab内容：                   │
│  - 传统先囤后约下单页内容            │
│                                     │
└─────────────────────────────────────┘
```

---

## Tab切换交互

### 从先囤后约切换到立即预订
1. 唤起日历页浮层（通兑则唤起选酒店浮层）
2. 默认选中第一个SKU、第一个房型
3. 若用户关闭浮层 → 切回先囤后约Tab
4. 若用户选择日期点击下一步 → 数据带回立即预订Tab

### 从立即预订切换到先囤后约
- 页面信息复制（入住人、联系手机号等）
- 保持立即预订的购买份数、SKU选择
