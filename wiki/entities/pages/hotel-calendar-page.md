# 酒店日历页

## 概述
酒店日历页（小搜detail页）是展示酒店日期和价格的页面，同时透出套餐模块为用户提供套餐购买选择。页面上接搜索结果页，下接套餐详情页。

## 核心属性

| 属性 | 值 |
|------|-----|
| 页面定位 | 日历展示 + 套餐推荐 |
| 核心目标 | 提升套餐售卖转化率 |
| 上游页面 | 搜索结果页、酒店列表页 |
| 下游页面 | 套餐详情页 |

---

## 套餐模块透出位置

酒店日历页在以下两个位置透出套餐模块：

### 1. 置顶套餐
- 位置：页面顶部
- 内容：单个精选套餐
- 目的：优先展示高转化套餐
- 详见：[[置顶套餐模块]]

### 2. 套餐推荐模块
- 位置：页面中部/下部
- 内容：多个套餐列表，分模块展示
- 目的：提供多样化套餐选择
- 详见：[[套餐推荐模块]]

---

## 套餐模块透出优先级

```
待预约套餐 > 置顶套餐 > G35 > 酒x > 日历房最低价
```

| 优先级 | 模块 | 说明 |
|--------|------|------|
| 1 | 待预约套餐 | 用户已有待预约订单 |
| 2 | **置顶套餐** | 圈品池配置的优选套餐 |
| 3 | G35 | - |
| 4 | 酒x | - |
| 5 | 日历房最低价 | - |

## 业务术语

| 术语 | 说明 |
|------|------|
| **小搜** | 酒店日历域的开发同学（负责detail页套餐模块） |
| **ictag** | 商品标签，用于业务逻辑识别 |
| **shid** | 酒店ID |
| **cico** | check-in check-out，入住离店时间 |
| **srid** | 房型rate ID |
| **nop** | 人数（number of people） |
| **裸房** | 不含X元素的套餐 |
| **含X** | 含优质X元素的套餐 |
