# 即买即约双Tab下单页

## 概述
即买即约双Tab下单页是支持即买即约商品的混合下单页，用户可以：
- 选择"立即预订"：即买即约模式，购买时直接选择入住日期
- 选择"先囤后约"：传统模式，购买凭证后预约

## 核心特征

| 属性 | 值 |
|------|-----|
| 订单生成 | 立即预订：2笔（套餐+日历）；先囤后约：1笔 |
| 支付方式 | 立即预订：合并支付；先囤后约：单独支付 |
| 库存锁定 | 立即预订：购买时直接锁定库存 |
| 适用人群 | 即时决策型 + 长期规划型 |

## 入口

- 即买即约日历页（下一步）
- 商品详情页（购买按钮进入先囤后约Tab）

---

## 页面结构

### Tab栏

```
┌─────────────────────────────────────┐
│  [立即预订]    |    [先囤后约]       │
│  先约先得，锁定库存  |  想住在约，不约可退 │
└─────────────────────────────────────┘
```

## 立即预订Tab

### 入口
- 即买即约日历页（下一步）
- 先囤后约Tab切换时唤起日历浮层

### 页面模块

以套餐下单页为框架，补充日历组件信息：

#### 1. 酒店信息模块
- 入离日期（点击可返回日历页修改）
- 酒店名称
- 套餐SKU名称
- x元素
- 不可退改政策

#### 2. 入住人组件（3期支持）
- 默认回填预订组件的入住人、手机号信息
- 国际场景增加电子邮箱

#### 3. 预订信息
- 房间数（点击可返回日历页修改）
- 入住人（支持点击icon后从入住人中选）
- 手机号
- 到店时间

#### 4. 会员注册组件
- 在套餐宝贝下单链路屏蔽会员注册
- 在日历房下单链路如果是会员专享RP或宝贝展示注册组件
- 支持异步注册开关

#### 5. 当单可享组件
- 合并套餐单+日历单的优惠信息
- 立即预订专属权益展示：
  - 立减x元
  - 免费改期
  - 免费升房
  - 实物权益

#### 6. 价格bar
```
总计 = sku价格 * 份数 - 优惠价格 + 补差价格

├── 套餐金额：sku券后价 * 份数
│   └── 展示SKU名称
├── 补差金额（若日期有补差才展示）
│   └── xx月xx日 热门日期房费补差
└── 优惠：套餐订单的优惠
    └── 展示具体优惠标签和明细
```

**国际场景补充**：
- 套餐金额新增外币金额
- 税费：到店付税费单独展示；在线付税费计入补差金额

### 特殊交互

#### 立减优惠滚动
进入即买即约下单页，判断当单可享受立减优惠，则：
- 优惠组件金额、底部价格滚动
- 底部增加浮条："已为您立减xx元，此价格包含日期补差金额"

#### 返回挽留弹窗
立即预订Tab下返回上一步时：
- 若日期有权益：展示权益挽留弹窗
- 若日期无权益：展示库存紧张挽留弹窗

## 先囤后约Tab

### 入口
- 商品详情页 - 购买按钮
- 商品详情页加购浮层 - 立即购买
- 立即预订Tab切换

### 页面内容
同线上先囤后约下单页样式

---

## Tab切换交互

### 从先囤后约切换到立即预订
1. 唤起日历页浮层（通兑则唤起选酒店浮层）
2. 默认选中第一个SKU、第一个房型
3. 若用户关闭浮层 → 切回先囤后约Tab
4. 若用户选择日期点击下一步 → 数据带回立即预订Tab

### 从立即预订切换到先囤后约
- 购买份数保持
- SKU选择保持
- 入住人/联系人信息不做回填（0425更新）

---

## 订单与资金流

### 立即预订订单结构
- **订单数量**：2笔（套餐订单 + 日历订单）
- **资金流拆分**：
  - 套餐订单对应套餐金额（优惠金额用在套餐单）
  - 日历订单对应补差金额

### 特殊场景处理
- 套餐订单或日历订单任一创单失败/可见失败 → 下单失败，两单同步取消
- 待支付状态 → 合并支付或合并取消
