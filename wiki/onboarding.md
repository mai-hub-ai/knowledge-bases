## 核心知识点速览

### 业务语义
```
单店 vs 通兑
├── 单店: item关联1个shid
└── 通兑: item关联>=2个shid
```

### 类目分类
```
酒店套餐类目
├── 含房类目（包含住宿）
│   ├── 在线预约套餐 (50794037) ← 即买即约主力
│   ├── 境内酒店套餐 (201189402)
│   ├── 境外酒店套餐 (201188002)
│   └── 境外在线申请套餐 (202032304)
│
└── 含餐/服务类目（不包含住宿）
    ├── 酒店餐饮美食 (201214101)
    └── 酒店服务（新）(201214201)
```

### 下单页分类
```
下单页类型
├── 先囤后约下单页 — 传统模式，生成1笔订单
├── 即买即约双Tab下单页 — 新模式，生成2笔订单
├── 日历预订下单页 — 日历预订型商品
└── 境内外酒店套餐下单页 — 境外商品
```

### 酒店日历页套餐透出优先级
```
待预约套餐 > 置顶套餐 > G35 > 酒x > 日历房最低价
```

### 即买即约核心概念
- **准入规则**: 黑名单机制（默认支持在线预约在售的国内套餐商品）
- **订单生成**: 套餐订单 + 日历订单（2笔）
- **权益体系**: 免费改期、免费升房、立减优惠（仅即买即约链路）

## 产品领域说明

我是酒店套餐产品经理，负责以下产品流程：

**用户流程**：
```
频道页 → 酒店日历页 → 套餐详情页 → 套餐SKU浮层 → 下单页
```

**核心目标**：
促进用户购买酒店套餐，提升套餐详情页→下单用户的转化（D2O）

**复杂度来源**：
- 横向：多端差异（iOS/Android/H5/小程序）、页面内模块多版本历史逻辑
- 纵向：交易状态流转、资金逻辑
